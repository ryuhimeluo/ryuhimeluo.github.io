Redis（第一篇博客测试）

Redis【非关系型数据库】：

key-value数据库 【存储数据库的结构map：键值对的方式进行存储数据】

数据结构丰富 【String字符串、Hash、List列表、Set集合、zset有序集合】

持久化 【RDB、AOF】

    RDB：

        把数据以快照的形式保存在磁盘上

        在指定的时间间隔内将内存中的数据集快照写入磁盘

    AOF：

        redis会将每一个收到的写命令都通过write函数追加到文件中，通俗的理解就是日志记录



    主从备份：

        采用异步

        一个主redis可以含多个从redis

        主从复制对于主redis服务器来说是非阻塞的，意味着当从服务器在进行主从复制同步过程中，主redis仍然可以处理外界的访问请求,对于从redis服务器来说也是非阻塞的

        主从复制提高了redis服务的扩展性，避免单个redis服务器的读写访问压力过大的问题，同时也可以给为数据备份及冗余提供一种解决方案

        作用：减轻数据库压力，提高查询效率，快速响应前端的请求

        参数：

        daemonize ==> yes 【后台启动】

        port 【端口号】

        bind 127.0.0.1 【绑定ip】

        SMAPSHOTTING 【redis持久化，会将缓存里面的数据放入磁盘中保存】

        requirepass *** 【设置密码】

Redis的应用场景：

缓存 【数据查询、短连接、新闻内容、商品内容】（最多使用）

分布式集群架构中的session 分离

聊天室的在线好友列表

任务队列 【秒杀、抢购、12306】

应用排行榜

网站访问统计

数据过期处理 【可以精确到毫秒】

安装教程（linux）

安装 gcc 环境 【yum install gcc-c++】

下载 tar 包

解压 tar 包 【tar -zxf redis-3.0.0.tar.gz】

编译redis源码包

进入redis文件内 【cd redis-3.0.0】

安装 【make】

安装redis

安装并修改路径 【make install PREFIX=/usr/local/redis】

启动Redis（有两种）

前端启动（不建议）

直接运行 bin/redis-server（bin目录是在/usr/local/App/redis/bin） 【./redis-server】

前端启动的关闭：ctrl+c

后端启动

将redis 源码包中的redis.conf 配置文件复制到 redis的bin 目录下

修改redis.conf 文件 【将 daemonize 由no改为yes】

在bin 目录下执行启动命令 【./redis-server redis.conf】

安装图形化界面客户端

连接失败的处理方法

修改redis.conf 配置文件 【注释bind】


开放6379端口号：

iptables：

编辑iptables /etc/sysconfig/iptables

添加一行:-A RH-Firewall-1-INPUT -m state –state NEW -m tcp -p tcp –dport 6379 -j ACCEPT


重启防火墙

/etc/init.d/iptables restart

防火墙操作（上面可能存在不可用情况，因为新版本缘故）

添加： firewall-cmd --zone=public --add-port=6379/tcp --permanent （--permanent永久生效，没有此参数重启后失效）

重新载入： firewall-cmd --reload

查看： firewall-cmd --zone= public --query-port=6379/tcp

删除： firewall-cmd --zone= public --remove-port=6379/tcp --permanent
